# BankGuard: Система мониторинга и оперативного уведомления о подозрительных транзакциях 

Проект разработан в рамках **Заключительного этапа Национальной технологической олимпиады (НТО) 2024/25** по профилю «Программная инженерия в финансовых технологиях».

## 📌 Обзор проекта
Данная система предназначена для защиты банковских клиентов от телефонного мошенничества и методов социальной инженерии. Программный комплекс объединяет данные о транзакциях, логах звонков и активности в смежных сервисах для выявления аномальных цепочек действий в реальном времени.

Система автоматизирует работу службы безопасности, заменяя ручной мониторинг автоматизированным анализом связей между различными источниками данных и обеспечивая мгновенное оповещение операторов через веб-интерфейс и мессенджер Telegram.

## ⚙️ Основные возможности
*   **Централизованная агрегация данных:** ETL-процессы по импорту и нормализации разрозненных банковских данных (логи звонков, транзакции, данные маркетплейсов) в единую структуру.
*   **Аналитический движок на основе связей:** Выявление подозрительных операций путем кросс-табличного анализа (например, сопоставление времени входящего вызова с мобильного устройства и последующего крупного перевода).
*   **Панель управления оператора (Dashboard):** Веб-интерфейс для мониторинга инцидентов в реальном времени с возможностью фильтрации, просмотра истории клиента и детальной информации по транзакциям.
*   **Система мгновенных уведомлений:** Интеграция с Telegram для автоматической рассылки алертов о критических инцидентах сотрудникам Департамента безопасности.
*   ** Feedback Loop:** Реализация механизма управления статусами инцидентов (подтверждение мошенничества / отклонение), позволяющая оператору оперативно реагировать на подозрительные события.

## 🏗 Архитектура системы
Проект построен по модульному принципу и включает следующие слои:

1.  **Database Layer:** Реляционное хранилище данных, обеспечивающее целостность информации о клиентах, их транзакциях и внешних коммуникациях.
2.  **Analysis Logic:** Модуль бизнес-логики, реализующий детерминированные алгоритмы проверки транзакций на основе SQL-запросов и серверных скриптов.
3.  **Backend API:** Серверный слой, обрабатывающий запросы от интерфейса и координирующий взаимодействие между базой данных и модулем уведомлений.
4.  **Frontend App:** Клиентское приложение для сотрудников колл-центра, визуализирующее данные об угрозах в удобном для анализа виде.
5.  **Notification Module:** Асинхронный сервис, отвечающий за взаимодействие с Telegram Bot API.
